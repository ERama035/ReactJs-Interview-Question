error handling in react :
---------------------------------------------
we have to use Erroboundry

Errors during rendering	Error Boundary (state-based fallback)
Errors in event handlers	try/catch + useState
Errors in async code (fetch, promises)	try/catch + useState
Errors from external libraries	Wrap with Error Boundary

If your goal is state-driven error handling, combine:
Error Boundaries → for rendering errors
Local state (useState) → for async/event errors

This gives you full coverage and keeps your UI stable and predictable.
